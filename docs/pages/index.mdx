# Traceway

Traceway is an open-source error tracking and monitoring platform for Go applications. It provides real-time visibility into your application's health through:

- **Exception Tracking**: Capture and group errors with full stack traces
- **Endpoint Monitoring**: Track HTTP request performance with P50/P95/P99 percentiles
- **Background Tasks**: Monitor background job execution and failures
- **System Metrics**: Collect CPU, memory, and Go runtime statistics

## Architecture

Traceway consists of three main components:

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│  Your Go App    │────▶│  Traceway API   │◀────│   Dashboard     │
│  + SDK Client   │     │   (Backend)     │     │   (Frontend)    │
└─────────────────┘     └────────┬────────┘     └─────────────────┘
                                 │
                    ┌────────────┴────────────┐
                    ▼                         ▼
            ┌───────────────┐         ┌────────────────┐
            │  ClickHouse   │         │   PostgreSQL   │
            │  (Telemetry)  │         │   (Accounts)   │
            └───────────────┘         └────────────────┘
```

## Features

### For Developers

- **Automatic Error Capture**: Panics and errors are automatically captured with full context
- **Request Scoping**: Add custom tags (user ID, tenant, etc.) to correlate errors
- **Segment Tracking**: Measure sub-operations within requests (database queries, API calls)
- **Background Task Support**: Track non-HTTP operations with the same tooling

### For Operations

- **Real-time Monitoring**: See errors and performance issues as they happen
- **Smart Grouping**: Similar errors are automatically grouped by normalized stack traces
- **Historical Analysis**: Query data across custom time ranges
- **Multi-project Support**: Manage multiple applications from a single dashboard
